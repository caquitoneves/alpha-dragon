# Architecture Overview

## Architectural Style

- DDD
- Clean Architecture
- Hexagonal Architecture
- Modular Monolith for MVP

## Architecture Principles

- The domain must remain free of infrastructure and framework coupling.
- CAD tools are adapters, not the source of truth.
- The core model must be parametric and extensible.
- Boundaries should be explicit and stable.

## Guiding Structure

- Domain
- Application
- Infrastructure
- Interfaces

## Platform Direction

The system should be capable of supporting:
- CAD adapters,
- intent engines,
- knowledge layers,
- simulation layers,
- public APIs and SDKs.
