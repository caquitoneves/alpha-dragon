# Domain Documentation

## Purpose

This section captures the language and concepts used to reason about the Alpha Dragon platform.

## Core Domain Concepts

- Engineering Project
- Assembly
- Part
- Body
- Feature
- Sketch
- Parameter
- Validation
- History
- Plugin

## Ubiquitous Language

The terms below should be used consistently throughout the codebase and documentation:
- Project: a top-level engineering container.
- Feature: a design operation that changes the model.
- Parameter: a named value controlled by rules or expressions.
- Snapshot: a versioned state of the model.
- Adapter: a connector to an external CAD or export ecosystem.
