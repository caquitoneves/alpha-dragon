# Engineering Principles

## 1. Domain First

Every major capability must start from the domain model before infrastructure or interfaces are added.

## 2. Platform Thinking

The project must be designed as a platform from day one, with clear extension points for adapters, plugins, knowledge and AI workflows.

## 3. Deterministic Core

The core engine must be deterministic, explainable and testable.

## 4. CAD Neutrality

No CAD tool is the source of truth. The universal engineering model is the source of truth.

## 5. Extensible Architecture

The architecture must support future modules without forcing rewrites of the core.

## 6. Documentation as Engineering Asset

Architecture, decisions and domain language are first-class engineering artifacts.
